## SEO 策略核心

不要抢“百科”这种大词。Offpedia 应该打高意图长尾词：

- Obsidian GitHub workflow
- Obsidian writing setup
- local-first knowledge base
- Notion offline alternative
- self-hosted knowledge base
- publish Obsidian vault
- Markdown writing workflow

中文可以打：

- Obsidian GitHub 写作工作流
- Obsidian 发布成网站
- 本地优先知识库
- Notion 离线替代
- 自托管知识库
- Markdown 写作系统

## 页面标题模板

### Stack

```txt
{场景}栈：{工具 A} + {工具 B} | Offpedia
```

### Kit

```txt
{Kit 名称}：可下载的 {场景} 模板 | Offpedia
```

### Wiki

```txt
{工具/概念} 是什么？本地优先工作流中的作用 | Offpedia
```

### Guide

```txt
如何 {完成结果}：{工具} 完整步骤 | Offpedia
```

### Compare

```txt
{A} vs {B}：怎么选？适合谁？| Offpedia
```

## 内链规则

每篇 Stack 至少链接：

- 2 个工具 Wiki
- 2 个概念 Wiki
- 1 个 Kit
- 1 个 Guide
- 1 个 Compare 或替代 Stack

每篇 Wiki 至少链接：

- 1 个 Stack
- 1 个 Guide 或 Kit

每篇 Kit 至少链接：

- 对应 Stack
- 对应 Guide
- 相关 Wiki

## 内容质量规则

避免：

- 泛泛介绍工具功能
- 抄官网
- 只列优缺点不讲场景
- 没有下一步 CTA
- 无具体字段和无结构

必须：

- 讲清适合谁
- 讲清不适合谁
- 给出推荐组合
- 给出可执行下一步
- 更新日期明确
