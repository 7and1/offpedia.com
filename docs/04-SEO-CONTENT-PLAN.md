# SEO Content Plan

## Strategy

Offpedia should target high-intent long-tail searches rather than broad encyclopedia terms.

Example query patterns:

- Obsidian GitHub writing workflow
- Obsidian publish website
- local-first knowledge base
- Notion offline alternative
- self-hosted knowledge base
- Markdown writing system

## Content Clusters

1. Obsidian writing workflows
2. Personal wiki publishing
3. Research knowledge systems
4. Self-hosted knowledge bases
5. Tool comparisons

## Page Templates

Stack title:

```txt
{Persona or use case} Stack: {Tool A} + {Tool B} | Offpedia
```

Kit title:

```txt
{Kit name}: downloadable {use case} template | Offpedia
```

Wiki title:

```txt
{Tool or concept}: role in local-first workflows | Offpedia
```

Guide title:

```txt
How to {complete task}: {tool} setup guide | Offpedia
```

Compare title:

```txt
{A} vs {B}: which should you choose? | Offpedia
```

## Internal Linking Rules

Each stack should link to:

- At least two tool wiki entries
- At least one concept wiki entry
- At least one kit
- At least one guide

Each wiki entry should link to a relevant stack, guide, or kit.

Each kit should link to the stack and guide that make it usable.

## Quality Rules

Avoid generic feature lists. Every page should explain who the workflow fits, who it does not fit, what to do next, and why the recommendation exists.
