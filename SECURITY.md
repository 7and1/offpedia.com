# Security Policy

## Reporting a vulnerability

Do not open a public issue with exploit details.

Preferred path:

1. Use GitHub private vulnerability reporting for this repository when that surface is enabled.
2. If private reporting is unavailable, open a minimal issue that requests a private contact route without posting technical details publicly.

## What to include

- Affected route, script, or workflow surface
- Reproduction steps
- Impact summary
- Any mitigation already tested

## Scope

This repository is a static content site. Typical reports may involve:

- build or deployment workflow leaks
- unsafe download or asset handling
- client-side XSS in rendered content
- exposure of internal operational details in public docs

Availability issues, factual content corrections, and SEO problems are not security reports.
